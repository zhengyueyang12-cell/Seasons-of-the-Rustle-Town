# Seasons of the Rustle Town — 开发指南

> 面向第一次接触本项目的开发者。引擎：**Godot 4.7** · 主场景：`Scenes/main.tscn`

---

## 一、项目概览

这是一款 2D 像素风农场生存游戏原型，核心循环包括：

- **移动与交互** — WASD 移动，快捷栏切换工具/武器/种子
- **耕地与种植** — 锄头翻地，种子播种，作物按天生长
- **砍树与生态** — 斧头砍树、掉落物、树桩再生、农场区内自动扩种
- **时间与天气** — 游戏内时钟、四季更替、晴雨切换（雨效粒子）
- **背包与快捷栏** — 24 格背包 + 9 格快捷栏，支持拖拽交换

架构原则：**数据用 `.tres` 资源定义，逻辑用 Autoload 单例调度，场景只负责摆放与引用。**

## 二、目录结构与脚本作用

### 2.2 主场景节点（`Scenes/main.tscn`）

```
Main (game_manager.gd)          ← 开局发物品、接时间/状态到 UI
└── World
    ├── TileDestructor          ← 锄头/镐翻土、破坏草地
    ├── FarmZone (Area2D)       ← 可种植区域，组名 farm_zone
    ├── TileMapLayerGround      ← 农场地形（terrain_map_bootstrap.gd）
    ├── YSort_Objects
    │   ├── Items/              ← 场景内静态拾取物
    │   ├── Drops/              ← 运行时掉落物父节点
    │   ├── Trees/              ← 预置树木
    │   ├── Crops/              ← 运行时作物父节点
    │   └── Player              ← 玩家与全套控制器
    ├── Camera2D                ← 跟随玩家（camera_follow.gd）
    ├── RainEffect              ← 地图固定雨效（见 2.6）
    └── Extends/CampFire        ← 营火装饰
└── CanvasLayer_UI (ui_layer.gd) ← 背包、快捷栏、HUD
```

### 2.7 物品与掉落

| 脚本 | 作用 |
|------|------|
| `item_drop.gd` | 掉落物物理弹跳 + 磁吸到玩家 `PickupAnchor` 后入背包 |
| `inventory_slot_ui.gd` | 背包/快捷栏格子 UI：显示、高亮、拖拽 |
| `item_slot_data.gd` | 单格数据（物品 id + 数量） |

### 2.8 天气 / 雨效（`systems/weather/rain_effect.gd`）

- 使用 **GPUParticles2D**：雨丝 + 水花 **sub-emitter**（雨滴消失时溅起水花）
- 纹理从 `art/items/TileSheets/rain..png` 四帧图集用 `AtlasTexture` 裁切（避免单独 png 未导入报错）
- 雨固定在**地图世界坐标**（`World/RainEffect`，`local_coords = false`），镜头移动时雨留在地图上
- 雨丝贴图本身向左下倾斜，**不要**开启 `ALIGN_Y_TO_VELOCITY`；用 `rain_wind_x` 调节风向

## 三、如何操作（游玩与调试）

### 3.1 运行项目

1. 用 **Godot 4.7** 打开项目根目录（含 `project.godot` 的文件夹）
2. 按 **F5** 运行，主场景为 `Scenes/main.tscn`
3. 首次打开会自动为 `art/` 下图集生成 `.import`（已在 `.gitignore` 中忽略）

### 3.2 按键一览

| 按键 | 功能 |
|------|------|
| **WASD** | 移动 |
| **1–9** | 切换快捷栏 |
| **鼠标滚轮** | 快捷栏前后切换 |
| **左键** | 攻击 / 使用工具 / 在耕地上播种 |
| **E** | 收获成熟作物 / 种植树苗（需满足条件） |
| **Tab** | 开关背包面板 |

### 3.3 农作流程示例

1. 快捷栏选中 **木锄** → 左键点击草地 → 变为耕地
2. 快捷栏选中 **种子** → 左键点击耕地 → 播种
3. 等待游戏内天数推进（或调 `TimeManager` 加速）→ 作物长高
4. 走近成熟作物按 **E** → 收获产物入背包

### 3.4 砍树流程

1. 快捷栏选中 **木斧**
2. 面朝树木，左键攻击
3. 树倒后掉落木材/树液等，磁吸拾取

### 3.5 雨效调试

在场景树选中 `World/RainEffect`，检查器可调：

| 属性 | 说明 |
|------|------|
| `force_rain_in_game` | `true` = 始终下雨（调试）；`false` = 跟随 `TimeManager` 天气 |
| `rain_amount` | 雨滴数量 |
| `emission_half_size` | 降雨区域半宽（地图坐标） |
| `rain_wind_x` | 风向（负值向左，需与雨丝贴图倾斜一致） |

控制台切换天气：`TimeManager.set_weather(TimeManager.Weather.RAINY)`

### 3.6 地形编辑器工具

选中 `TileMapLayerGround` 节点：

- **重置为草地基底** — 整层铺草
- **刷新全部地形融合** — 重算 Terrain 邻接融合

若修改了春季地形图集，在编辑器 **文件 → 运行** 执行：

`systems/terrain_system/build_farm_terrain_tileset.gd`

以重新生成 `resources/tilesets/farm_ground.tres`。

## 四、如何扩展

### 4.1 添加新作物

1. 在 `resources/crops/` 新建 `my_crop.tres`（类型 `CropResource`）
2. 在 `resources/items/seeds/` 新建种子 `ItemResource`
3. 在 `resources/items/produce/` 新建产物
4. 确保 `crops..png` 图集有对应生长阶段帧
5. **无需改代码** — `CropRegistry` 启动时自动扫描 `resources/crops/`

### 4.2 添加新树种

1. 在 `resources/trees/` 新建 `.tres`（类型 `TreeResource`）
2. 配置四季图集路径与生长阶段
3. 在场景中实例化 `Scenes/props/nature/tree.tscn` 并覆盖 `tree_resource`
4. `TreeRegistry` 自动注册

### 4.3 添加新工具 / 武器

1. 在 `resources/tools/` 或武器目录新建 `.tres`
2. 设置 `id`、`icon`、`use_group` 等字段
3. `ToolRegistry` / `WeaponRegistry` 启动时注册到 `ItemRegistry`
4. 若需新行为，在 `tool_controller.gd` 或 `tile_destructor.gd` 增加分支

### 4.4 添加新物品（纯拾取 / 合成）

1. 新建 `resources/items/xxx.tres`
2. 放入 `resources/items/` 子目录
3. `ItemRegistry` 按文件名自动加载

### 4.5 扩展雨效 / 其他天气

- 修改 `rain_effect.gd` 中粒子参数，或替换 `rain..png` 图集
- 在 `TimeManager.weather_changed` 信号里切换不同 VFX 场景
- 可参考现有结构新建 `snow_effect.gd`，挂到 `World` 下固定坐标

### 4.6 扩展 UI

- 背包格：`Scenes/slot.tscn` + `inventory_slot_ui.gd`
- 快捷栏格：`Scenes/hotbar_slot.tscn`（激活高亮样式见 `resources/ui/hotbar_slot_active.tres`）
- 全局 HUD 刷新走 `UIManager` 信号，避免 UI 脚本直接读游戏状态

### 4.7 常用约定

- **组名**：`tile_destructor`、`farm_zone`、`tree`、`choppable_tree`、`tree_stump`
- **查找节点**：优先 Autoload；必须找场景节点时用 `get_tree().get_first_node_in_group()` 或固定路径 `World/YSort_Objects/Crops`
- **像素风**：项目已设 `canvas_texture_filter=Nearest`，新贴图保持一致

## 六、常见问题

**Q: `preload("xxx.png")` 报 unrecognized file extension？**  
A: 确保文件扩展名正常（单 `.png`），且已在编辑器中打开过项目让 Godot 生成 import。雨效统一用 `rain..png` + `AtlasTexture` 裁切。

**Q: 快捷栏高亮编辑器可见、游戏里不可见？**  
A: 高亮由 `inventory_slot_ui.gd` 的 `bind_slot()` 在运行时绑定，确认 `ui_layer.gd` 已调用各格的 `bind_slot()`。

**Q: TileMap 格子变粉/消失？**  
A: 运行 `build_farm_terrain_tileset.gd` 重建 TileSet，再在 `TileMapLayerGround` 点「刷新全部地形融合」。

**Q: 雨的方向和雨丝贴图不一致？**  
A: 贴图已带倾斜，保持 `ALIGN_Y_TO_VELOCITY` 关闭，调节 `rain_wind_x`（默认 `-0.2`）。
